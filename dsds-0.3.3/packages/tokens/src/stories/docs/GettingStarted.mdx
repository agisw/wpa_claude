# 시작하기

## DS 디자인 시스템 소개

Figma 디자인 토큰 기반으로 자동빌드되는 공통 CSS 파일, 이미지, 아이콘등을 패키지화 했습니다.

## 패키지 구조

`packages/tokens`는 모노레포에서 디자인 토큰만을 라이브러리화해서 제공하는 패키지입니다. `packages/tokens/src/styles` 폴더의 파일들이 디자인 토큰 라이브러리를 구성합니다.

- `packages/tokens/src/styles/variables`: primitive와 semantic 토큰들이 포함되어 있습니다.
- `packages/tokens/src/styles/*.css`: 컴포넌트 토큰들입니다.

다음은 `packages/tokens/src/styles` 폴더 내 파일 및 폴더 목록입니다:

| 파일/폴더 | 설명 |
|------------|------|
| variables | primitive와 semantic 토큰들이 포함된 폴더 |
| badge.css | 컴포넌트 토큰 (badge 컴포넌트용) |
| boxes.css | 컴포넌트 토큰 (boxes 컴포넌트용) |
| button.css | 컴포넌트 토큰 (button 컴포넌트용) |
| checkbox.css | 컴포넌트 토큰 (checkbox 컴포넌트용) |
| dropdown.css | 컴포넌트 토큰 (dropdown 컴포넌트용) |
| header.css | 컴포넌트 토큰 (header 컴포넌트용) |
| radio.css | 컴포넌트 토큰 (radio 컴포넌트용) |
| tabs.css | 컴포넌트 토큰 (tabs 컴포넌트용) |
| toast.css | 컴포넌트 토큰 (toast 컴포넌트용) |

- 스토리북: https://dsds.mwebdev.samsungds.net/storybooks/tokens

## 참고: Figma 디자인 토큰 자동빌드 방법

- Figma 에서 사용하는 Styles 과 Variables 를 Tokens Studio 플러그인으로 Import 후을 사내 Github의 `design/tokens.json` 으로 Push 합니다.
- Github Actions 의 CI/CD 스크립트가 실행됩니다.
  1. `design/tokens.json` 파일을 `global.css`, `light.css` 로 transform 후
  2. 관련 CSS 변경사항이 있을 경우 빌드된 CSS 파일까지 자동 Commit 합니다.
- 변경된 CSS를 반영한 Storybook을 자동 빌드 배포됩니다.
